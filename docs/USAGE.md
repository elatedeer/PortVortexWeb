# 使用说明

这份文档面向最终用户和日常维护人员，说明 PortVortex Web Flasher 各页面的用途、基本操作和注意事项。

## 1. 启动方式

### 开发模式

~~~bash
npm install
npm run dev
~~~

开发模式会同时启动前端和后端。

### 生产模式

~~~bash
npm run build
npm start
~~~

生产模式下，后端会优先托管 dist/ 中的前端页面。

## 2. 页面总览

### 下载页

用于固件下载和配置导入。

可做的事情：

- 选择下载模式
- 输入设备 Token
- 上传固件文件
- 选择目标芯片
- 设置基地址、擦除方式、连接方式
- 打开专家模式调整更细的参数

### 实时对话页

用于与设备进行实时通信。

包含三个通道：

- UART1
- RS485
- CAN

每个通道都支持：

- 连接 / 断开
- 文本发送
- 文件发送
- 收发格式切换
- 定时发送
- 快捷短语

### 设备分组页

用于维护设备清单和批量升级。

可做的事情：

- 新建分组
- 导入 Token
- 导入设备认证数据
- 导出设备表格
- 对某个分组批量下发升级任务

### Token 历史页

用于查看和复用最近使用过的设备 Token。

可做的事情：

- 查看最后使用时间
- 查看使用次数
- 回填到当前输入框
- 删除单条记录
- 清空全部记录

### AI 助手页

用于配置第三方 AI 接口并进行对话。

支持：

- 配置 API URL
- 选择模型
- 切换直接对话模式
- 将串口上下文作为对话参考

### 反馈页

用于提交问题、建议或其他反馈。

可附带：

- 标题
- 联系方式
- 描述
- 图片或附件

### 日志页

用于查看系统与通信相关日志，方便排障。

## 3. 下载页使用流程

1. 输入设备 Token。
2. 选择目标芯片。
3. 上传 BIN 或 HEX 固件。
4. 按需要设置：
   - aseAddr
   - 擦除方式
   - attach 策略
   - 是否自动复位
   - 是否启用单包/离线模式
5. 点击开始。
6. 在下载日志中查看过程。

### 专家模式

专家模式适合有明确调试需求的人员，里面可以调：

- 传输窗口
- chunk 大小
- chunk 延迟
- ACK 超时
- UART bootloader 相关参数
- 自定义算法文件
- 芯片配置导入

如果你没有明确需求，建议先保持默认值。

## 4. 实时对话页使用流程

1. 输入设备 Token。
2. 选择通道类型。
3. 必要时打开串口配置或 Modbus 配置。
4. 点击连接。
5. 在输入框里发送消息，或者直接上传文件发送。

### 格式说明

常用格式包括：

- ASCII
- UTF-8
- HEX

更多格式包括：

- UTF-16LE
- UTF-16BE
- Unicode
- GBK
- GB18030
- Base64

切换格式时，系统会尽量自动转换当前输入内容。

### 定时发送

勾选定时发送后，可设置间隔为毫秒或秒。

适合做：

- 心跳包
- 轮询命令
- 压测
- 自动采集

## 5. Modbus 配置

Modbus 主要用于 RS485 页面。

模式说明：

- ridge：透明桥接
- modbus_slave：从站
- modbus_master：主站

注意：

- 一旦切到 modbus_slave 或 modbus_master，RS485 透明对话会暂停。
- 主站模式下需要配置轮询任务。

## 6. 串口配置

串口配置当前覆盖：

- UART1
- RS485

可以调整：

- 波特率
- 数据位
- 校验位
- 停止位
- 流控

这些配置会保存在后端，并可刷新读取。

## 7. 设备认证与安全

设备认证信息支持两种来源：

- 环境变量
- device-auth.json

其中 device-auth.json 支持：

- 数组格式
- 映射格式

建议把设备密钥保存在受控位置，不要放进浏览器里。

## 8. 串口转发脚本

scripts/serial_forward.py 用于把本地串口桥接到 PortVortex 的实时通信通道。

### 常见用法

~~~bash
python scripts/serial_forward.py <device_id> COM3 --mode uart
~~~

使用 com0com 自动创建虚拟串口对：

~~~bash
python scripts/serial_forward.py <device_id> --mode rs485 --auto-com0com
~~~

如果需要指定本地服务地址：

~~~bash
python scripts/serial_forward.py <device_id> --server http://localhost:3000
~~~

### 依赖

- Python 3
- pyserial
- Windows 下建议准备 com0com

## 9. 芯片配置导入与导出

你可以通过后端接口或页面里的导入功能维护芯片配置：

- 导入单条芯片配置 JSON
- 导入芯片配置包
- 导入 CMSIS-Pack 相关数据
- 导出当前配置包

适合场景：

- 新增芯片型号
- 从 PACK 批量生成配置
- 在不同环境之间迁移配置

## 10. 常见问题

### 页面打开了，但烧录请求失败

先确认：

- 后端是否在 PORT 指定端口启动
- 前端是否能访问 http://localhost:3000
- 设备 Token 是否填写
- 固件文件是否已选择

### 反馈提交失败

先确认 SMTP 配置是否完整，尤其是：

- SMTP_USER
- SMTP_PASS

### 串口转发脚本无法创建虚拟串口

先确认：

- com0com 是否已安装
- 是否以管理员权限运行
- setupc.exe 是否在 PATH 中，或通过参数显式指定
